Windows 系统安装 Hexo 并本地跑起来的完整步骤

一、安装必备环境：Git + Node.js

1. 安装 Git

  1. 打开官网：https://git-scm.com/
  2. 下载 Windows 版本（64位），默认安装即可，一路 Next。
  3. 验证：打开 CMD / PowerShell，输入 bash git --version 能显示版本号就成功。

2. 安装 Node.js（含 npm）

  1. 官网：https://nodejs.org/
  2. 下载 LTS 长期支持版（推荐）。
  3. 安装时默认勾选 Add to PATH，一路 Next。
  4. 验证（重启 CMD/PowerShell 再输入）： bash node -v npm -v 都能输出版本号即可。

二、安装 Hexo 命令行工具 打开 CMD 或 PowerShell，

输入： bash npm install -g hexo-cli

  • -g 表示全局安装

  • 国内慢可以换淘宝镜像（可选）： bash npm config set registry https://registry.npmmirror.com

验证是否装好： bash hexo -v 输出版本信息即成功。

三、初始化博客项目

  1. 找一个盘（建议非中文路径），新建文件夹，例如： D:\hexo-blog
  2. 在文件夹里打开 CMD（或地址栏输入 cmd 回车），执行： bash hexo init 或指定目录： bash hexo init blog 等待依赖安装完成。
  3. 进入博客目录： bash cd D:\hexo-blog
  4. 安装项目依赖： bash npm install

四、本地启动博客

在博客目录下执行： bash hexo s # 或 hexo server

  • 看到 INFO Hexo is running at http://localhost:4000/ 就成功
  • 浏览器打开：http://localhost:4000/ 即可看到默认博客页面。

常用命令：

bash hexo clean # 清除缓存

hexo generate # 生成静态文件（简写 hexo g）

hexo server # 本地预览（简写 hexo s）

hexo deploy # 部署到服务器（需配置，简写 hexo d）

五、常见问题（Windows）

1. “hexo 不是内部或外部命令” 原因：环境变量没生效 解决： - 重启 CMD/PowerShell - 或手动把 C:\Users\你的用户名\AppData\Roaming\npm 加入系统 PATH。

2. npm 安装卡住/报错 - 换淘宝镜像： bash npm config set registry https://registry.npmmirror.com - 清理缓存： bash npm cache clean --force

3. 中文路径导致报错 不要放在桌面、中文用户名、中文文件夹下，一律用纯英文路径。
