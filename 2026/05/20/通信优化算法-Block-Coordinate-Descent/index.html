通信优化算法-Block Coordinate Descent

Posted on 2026-05-20

BCD 算法（Block Coordinate Descent，块坐标下降法）是凸优化和大规模机器学习中非常经典且高效的一种迭代算法。

当一个优化问题的变量非常多、维度极高，或者整体求解非常困难时，BCD 的核心思想是：“分而治之”。它将高维的变量划分为若干个独立的“块”（Blocks），每次只更新其中一个块的变量，而将其他所有块的变量视为常数。

1. 核心思想与数学表达

其中变量 $x$ 被划分为 $m$ 个块：$x = (x_1, x_2, \dots, x_m)$，每个 $x_i$ 可以是一个标量或一个高维向量。

标准的 Gauss-Seidel 轮询更新策略

在第 $k$ 次迭代中，算法按照固定的顺序（如从第 1 个块到第 $m$ 个块）依次交替优化：

核心点： 更新当前块 $x_i$ 时，已经更新过的块（$1 \le j < i$）使用最新值 $x_j^{k+1}$，还未更新的块（$i < j \le m$）使用旧值 $x_j^k$。

2. 块选择策略（Block Selection）

除了上述最常用的轮询（Cyclic / Gauss-Seidel）顺序外，还有其他选择块的方式：

  • 随机选择（Randomized BCD）： 每一步迭代时，从 $m$ 个块中随机抽取一个进行更新。在大规模机器学习中（如坐标下降法训练线性 SVM），随机策略往往比轮询策略收敛更快、性质更好。
  • 贪心选择（Gauss-Southwell）： 每次选择梯度（或偏导数）最大、或者能让目标函数下降最多的那个块进行更新。计算成本稍高，但步步见效。

3. 收敛性与凸优化中的挑战

BCD 算法虽然直观，但在理论收敛性上是有条件的。并不是所有的凸函数用 BCD 都能收敛到全局最优解。

能安全收敛的情况：

  1. 函数严格凸且连续可微： 如果 $f(x)$ 是光滑的强凸/严格凸函数，BCD 能够稳步收敛到全局最优解。
  2. 可分非光滑项： 目标函数形如 $F(x) = f(x) + \sum_{i=1}^m g_i(x_i)$，其中 $f$ 是光滑凸函数，而 $g_i$ 是非光滑凸函数（例如 $L_1$ 正则化项 $|x|_1$）。此时配合近端梯度（Proximal）的 BCD 依然可以收敛。

无法收敛的“大坑”（反例）：

如果非光滑项不可分，BCD 可能会陷入死锁（Stall）。

  • 经典反例： 目标函数带有形如 $|x_1 - x_2|$ 的项（其等高线有尖锐的“V”字形槽）。如果当前点位于槽谷，无论沿着 $x_1$ 轴还是 $x_2$ 轴单独移动，目标函数值都会上升，导致 BCD 算法卡在原地，而该点根本不是极值点。

4. BCD 的优缺点

优点

  • 降低计算复杂度： 将一个昂贵的高维优化问题拆解为多个廉价的低维子问题。
  • 内存友好： 每次只需要加载和处理一部分数据/变量，非常适合大数据场景。
  • 可利用闭式解（Closed-form Solution）： 很多时候，拆分后的子问题具有解析解（例如最小二乘、软阈值算子），更新极其迅速。

缺点

  • 串行依赖： 标准的 Gauss-Seidel BCD 必须等前一个块更新完才能更新下一个，不易大规模并行化（针对这一点，催生了异步坐标下降算法）。
  • 对非光滑非可分函数无能为力。

5. 典型应用场景

  • LASSO 回归 / $L_1$ 正则化： 每次只优化一个权重参数，子问题有闭式解（软阈值），这是 BCD 的绝佳舞台（坐标下降法）。
  • 矩阵分解（Matrix Factorization / NMF）： 固定左矩阵优化右矩阵，再固定右矩阵优化左矩阵，退化为交替最小二乘法（ALS，这是 BCD 只有两块变量时的特例）。
  • 机器学习中的多块优化： 许多带有结构化约束的问题，在引入拉格朗日乘子后，常使用基于 BCD 思想的 ADMM（交替方向乘子法） 求解。
